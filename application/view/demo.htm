What the ...?

PHP Version

PHPCanvas uses strict object-oriented programming and some required functions were only made available later than PHP 4.

Integration type of system framework

Centralized integration is where the framework, the folder labelled phpcanvas, is located on a different location outside the application. This makes the framework reusable by other PHPCanvas aapplications. Also, centralizing increases security and reliability to the framework when working with multiple applications.

Localized integration prioritizes the portability of an application. This ensures the application will work on a specific version of PHPCanvas. Because the framework is located inside the application, this makes the migration from one server to another a lot less tedious.

Apache mod_rewrite

PHPCanvas makes URLs more readable with the help of Apache's mod_rewrite which is set in Apache's configuration file. Some commercial server's disable the access and checking of the configuration file.

If mod_rewrite is disabled, moving around the application can be simulated by passing the controller, method and parameters as a query string.

                example:
                from: http://mysite.com/controller/method/param1/param2
                to: http://mysite.com/?controller/method/param1/param2

Temporary Folder

PHPCanvas uses a temporary folder where it stores logs, caches and any other temporary files. Check the read/write permission for this folder before running the application especially for servers running on Linux or Unix.

Load fallback function

PHPCanvas detects if the server supports new but required functions like json_encode and json_decode. If not, it creates synonymous functions to replace them. It is still advised to run the framework in PHP not older than 5.3.

Database Connection

This may not be required in initial setup or not at all. PHPCanvas Tries to connect using to MYSQL database using the first credential entry in database.ini. If you are going to use a database, it is advised to test it here first.

site_url configuration in config.ini

This is an optional configuration. Setting up the site_url with your absolute site location eases the referrence to the site in development and file access in html using the base tag.
